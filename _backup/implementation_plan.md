# QA 자동화 도입 로드맵 가이드

안티그래비티(AntiGravity)와 함께하는 QA 자동화 도입을 위한 단계별 가이드입니다.

## 1. 현황 파악 및 목표 설정 (Assessment & Goal)
QA 자동화를 시작하기 전, 현재 상황에 맞는 목표를 정하는 것이 중요합니다.
- **대상 프로젝트**: 어떤 프로젝트에 먼저 도입할 것인가? (신규 vs 레거시)
- **우선 순위**: 가장 빈번하게 발생하는 버그 유형이나, 테스트 비용이 높은 수동 테스트 구간 식별 (예: 회원가입, 결제 플로우)

## 2. 테스팅 전략 수립 (Testing Strategy)
프론트엔드 환경에서 효율적인 테스트 피라미드를 구성합니다.
- **E2E 테스트 (End-to-End)**: 사용자의 관점에서 브라우저 상의 전체 흐름 테스트 (안티그래비티 강점)
  - *추천 도구*: Playwright (최신, 빠름, 안정적) 또는 Cypress
- **통합/유닛 테스트 (Integration/Unit)**: 컴포넌트 및 로직 단위 검증
  - *추천 도구*: Vitest (Vite 기반), Jest

## 3. 안티그래비티 활용 방안 (How AntiGravity Helps)
안티그래비티는 다음과 같은 작업을 수행하여 자동화 도입을 가속화합니다:
- **테스트 케이스 생성**: 기획서나 현재 코드를 분석하여 테스트 시나리오 제안
- **테스트 코드 작성**: Playwright/Vitest 등 설정 및 실제 테스트 코드 작성
- **UI 기반 테스트**: 브라우저를 직접 제어하여 시각적 회귀 테스트 수행
- **CI/CD 통합**: GitHub Actions 등을 통한 자동 실행 파이프라인 구성 지원

## 4. 실행 계획 (Action Plan)
가장 작은 단위부터 시작하여 점진적으로 확장하는 것을 권장합니다.

1. **POC (Proof of Concept) 환경 구성**:
   - 빈 프로젝트 또는 현재 프로젝트의 클론에 Playwright 설치
2. **핵심 기능 1개 자동화**:
   - 예: 로그인 페이지 로드 -> 유효한 계정 입력 -> 로그인 성공 확인
3. **결과 공유 및 피드백**:
   - 팀원들에게 실행 영상/리포트 공유하여 효용성 입증
