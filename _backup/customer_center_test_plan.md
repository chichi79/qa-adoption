# 고객센터 UI 개편 테스트 계획 (Pilot Plan)

## 📌 테스트 대상
1.  **메인 페이지 (`/help`)**: 신규 반응형 작업 (PC/Mobile)
2.  **메일 문의하기 (`/help/contact`)**: 기존 레거시 수정

## 🧪 테스트 시나리오 제안

### 1. 메인 페이지 (반응형 점검 중심)
화면 크기에 따라 레이아웃이 깨지지 않고 잘 바뀌는지 검증합니다.
- **CASE 1-1 (PC 뷰)**:
    - 넓은 화면(1280px 이상)에서 '자주 묻는 질문(FAQ)'이 가로 그리드로 잘 보이는가?
    - 상단 GNB 메뉴가 펼쳐져 있는가?
- **CASE 1-2 (Mobile 뷰)**:
    - 좁은 화면(375px)으로 줄였을 때, GNB가 '햄버거 버튼'으로 바뀌는가?
    - FAQ 리스트가 세로 일렬로 정렬되는가?
    - *👉 안티그래비티 TIP: 시각적 회귀 테스트(Screenshot)가 아주 효과적임!*

### 2. 메일 문의하기 (기능/레거시 점검 중심)
기존 기능을 건드렸으므로, 기능이 망가지지 않았는지가 핵심입니다.
- **CASE 2-1 (필수 입력값 검증)**:
    - 제목, 내용, 이메일을 비워두고 [보내기] 클릭 -> 경고 문구가 뜨는가?
- **CASE 2-2 (정상 접수 확인)**:
    - 모든 항목 입력 후 [보내기] 클릭 -> "접수되었습니다" 완료 팝업/페이지 확인.
- **CASE 2-3 (이메일 유효성)**:
    - 이메일에 `abc`라고만 쓰고 보내기 -> "이메일 형식이 올바르지 않습니다" 뜨는가?

## 🚀 실행 전략
1.  **식별자 심기**: 리팩토링 중인 코드에 `data-testid` 미리 확보. (예: `btn-contact-submit`)
2.  **모바일 테스트 코드**: Playwright의 `devices` 설정을 사용해 아이폰/갤럭시 해상도로 자동 테스트 수행.
