# QA 자동화 프로세스 시뮬레이션 가이드

이 문서는 우리가 구축한 자동화 테스트가 실제 개발 현장에서 버그를 어떻게 잡아내는지(QA)를 시뮬레이션하는 절차입니다.

## 시나리오: "개발자가 실수로 코드를 건드렸다!" 😱

### 1단계: 정상 상태 확인 (Green) ✅
- 현재 코드는 정상이고, 테스트는 통과했습니다.
- 이는 "배포 가능한 안전한 상태"를 의미합니다.

### 2단계: 버그 발생 (Red) ❌
- 개발자가 로그인 로직을 수정하다가 실수로 비밀번호 검증 로직을 바꿔버렸다고 가정해 봅시다.
- 우리는 `LoginForm.tsx`를 강제로 수정하여 버그를 심을 것입니다.

### 3단계: QA 테스트 수행 (Catch) 🕵️
- 자동화 테스트를 돌려서 이 실수가 검출되는지 확인합니다.
- 테스트가 실패(Fail)하면, QA 시스템이 정상 작동하여 버그 유출을 막은 것입니다.

### 4단계: 디버깅 및 수정 (Refactor) 🛠️
- 리포트를 통해 무엇이 잘못되었는지 파악하고 코드를 원복합니다.

### 5단계: 재검증 (Pass) 🔄
- 다시 테스트를 수행하여 녹색 불(Pass)이 들어오는지 확인합니다.
